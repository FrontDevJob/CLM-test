# Технические особенности реализации

## Презентацию открывать

`slides --> start --> index.html`

## 1. Отсутствие семантики в вёрстке

Вёрстка выполнена с прицелом на среду CLM-презентации. В таких проектах семантика не используется, так как:

- она не влияет на SEO;
- не нужна скринридерам (происходит прямой контакт с представителем);
- может конфликтовать с движком отображения слайдов.

**Основной фокус:**
- стабильность отображения;
- скорость перелистывания;
- скорость выполнения тестового задания.

## 2. Переход между слайдами

Переход осуществляется через возврат на стартовый слайд.  

Для больших проектов дополнительно можно было бы реализовать:
- функцию отслеживания свайпа по экрану;
- систему веток.

В рамках текущей задачи это лишнее.

## 3. Трекинг

Весь трекинг записывается в `localstorage`. Для просмотра в консоль браузера ввести: `JSON.parse(localStorage.getItem('clm_analytics'))`. Для очистки хранилища: `localStorage.clear('clm_analytics')`.

### userActions
Здесь указываются объекты, на которые нажимал пользователь, а также время нажатия.

### Отслеживание зажатия (вместо наведения)

Отслеживание **наведения** мыши не реализовано, но реализовано отслеживание времени **зажатия** мыши на объекте.

> Для отслеживания зажатия на нужные объекты необходимо заранее навесить атрибут `data-name` и указать название.

**Проверка функционала отслеживания зажатия:**  
Функцию можно проверить на слайде *«Описание действия»*. Атрибут присутствует на карточках:
- «Боль»;
- «Воспаление»;
- «Температура».

### topHovered
Содержит массив объектов с данными:
- об объекте, на котором происходило зажатие;
- общее время, сколько его зажимали (учитываются все зажатия на объекте, даже если их было несколько).

В этот массив входит **топ-10** таких объектов, отсортированных по общему времени зажатия.

## 4. Дублирование JSON-данных в js-файлах

В слайдах "Конкурентная группа" и "Объем продаж" нужно хранить данные в отдельном JSON. Если не пользоваться Live Server, то CORS будет ругаться и не даст подгрузить данные, из-за чего слайды окажутся пустыми. Чтобы это обойти на fetch был добавлен обработчик catch, который обрабатывает ошибку, вызванную CORS, и берет данные из fallbackData.
